# Documentation Policy

## Документация для нового кода

Для новых публичных сущностей и новых функций в проекте по умолчанию нужно добавлять KDoc.

## Язык документации

По умолчанию новый KDoc нужно писать на русском языке.

Практическое правило:

- использовать русский язык для новой встроенной документации, если пользователь не попросил иначе;
- не смешивать русский и английский без отдельной причины;
- для публичных `class`, `interface`, `enum`, `data class` и важных функций добавлять краткое описание назначения;
- для orchestration-функций и функций с параметрами при необходимости добавлять `@param` и `@return`.

## README и эксплуатационная документация

README проекта должен объяснять:

- как подготовить `Ollama`;
- как устроен `docs/`-корпус;
- как запускать индексацию;
- как сравнивать стратегии chunking;
- какие части входят в MVP, а какие оставлены на будущее.
