# Overview

Проект задуман как учебный локальный индексатор документов.

## Цель первой версии

Собрать понятный end-to-end пайплайн:

- локальные документы;
- извлечение текста;
- две стратегии chunking;
- embeddings через Ollama;
- сохранение индекса в SQLite;
- сравнение fixed и structured chunking.

## Архитектурное направление

- `SQLite-first` в первой версии;
- `FAISS-ready` как направление дальнейшего развития;
- без лишней сложности в MVP;
- с понятным разделением на CLI, config, loader, extractor, chunking, embedding, storage.

## CLI-ориентиры из day_20

- CLI parsing держать отдельно от orchestration и domain logic;
- пользовательский вывод делать через отдельный formatter;
- `main` оставлять тонкой точкой входа;
- настройки CLI и среды загружать до выполнения команд;
- для русскоязычного вывода на Windows явно настраивать UTF-8.
