# Agent Preflight

Короткая памятка перед началом нового блока изменений.

## Обязательная проверка

1. Прочитать локальный `MemoryBank` проекта.
2. Свериться с общим `MemoryBank`.
3. Проверить этот preflight-файл.
4. Перед новым блоком правок посмотреть текущее состояние рабочей папки.
5. Если в проекте уже есть незавершённые изменения и начинается новый смысловой блок с правками, напомнить про коммит и дождаться явного решения пользователя.

## Критичные правила проекта

- Первая версия проекта должна оставаться маленьким и понятным учебным reference.
- Реализуем только текущий MVP: каркас, загрузка, извлечение текста, chunking, embeddings, SQLite storage, сравнение стратегий.
- Поиск и интеграция с FAISS пока остаются future scope, но архитектурный задел допустим.
- Конфигурацию хранить в `application.conf`, а не зашивать в код.
- Для нового публичного кода и важного orchestration-кода по умолчанию добавлять документацию.
- Для CLI на Windows считать UTF-8 частью рабочего сценария, а не косметикой.

## Особенности текущего проекта

- Основной стек: Kotlin/JVM.
- Формат корпуса для первой версии: специально собранный учебный `docs/`-корпус.
- Хранилище MVP: только SQLite.
- CLI должен быть минималистичным и понятным, без перегруженного UX.
- Для ручного запуска предпочитать direct launcher из `build\install\local-document-indexer\bin\local-document-indexer.bat`, а не `gradlew run`.
- Фразу пользователя `собери проект` трактовать как запуск `powershell -ExecutionPolicy Bypass -File .\scripts\start-manual-check.ps1`.
- Фразу пользователя `запусти проект` трактовать как запуск `powershell -ExecutionPolicy Bypass -File .\scripts\start-manual-check.ps1 -SkipBuild`.
