# Сравнение plain и RAG

Итоговый отчёт по 10 контрольным вопросам из [control-questions.md](/C:/Users/compadre/Downloads/Projects/AiAdvent/day_22/docs/control-questions.md).

Полный сырой прогон был собран скриптом [run-rag-evaluation.ps1](/C:/Users/compadre/Downloads/Projects/AiAdvent/day_22/scripts/run-rag-evaluation.ps1) по корпусу:

- `docs/articles/doroshevich`

Это важно: для evaluation использовался именно корпус художественных текстов, без вспомогательных файлов из `docs/`, чтобы retrieval не подхватывал контрольные вопросы и служебные заметки как часть базы знаний.

## Методика

Для каждого вопроса были собраны три ответа:

- `plain` — внешний LLM без retrieval;
- `rag-fixed` — retrieval по `index-fixed.db`;
- `rag-structured` — retrieval по `index-structured.db`.

Параметры прогона:

- `topK = 3`
- генерация ответа через внешний LLM API
- embeddings и retrieval через `Ollama + SQLite + cosine similarity`

## Ключевые наблюдения

- `plain` часто даёт правдоподобные, но не привязанные к корпусу ответы.
- `rag` заметно дисциплинирует модель: когда контекст нерелевантен, модель чаще честно отказывается отвечать, а не фантазирует.
- `structured` в retrieval в среднем лучше `fixed`, но превосходство пока не радикальное.
- Основной узкий момент проекта сейчас не генерация, а retrieval: на части вопросов нужный источник не попадает в top-3 вообще или попадает не тем чанком.

Формальные сигналы по retrieval:

- `fixed`: ожидаемый источник в `top1` у `3/10` вопросов
- `fixed`: ожидаемый источник в `top3` у `5/10` вопросов
- `structured`: ожидаемый источник в `top1` у `5/10` вопросов
- `structured`: ожидаемый источник в `top3` у `5/10` вопросов

## Примеры различий

### Где `plain` хуже

- В вопросе про цвета Адена модель без retrieval выдумала палитру с серыми скалами, синим морем и зелёной водой, хотя в корпусе ключевой контраст строится вокруг чёрного и белого.
- В вопросе про самоли `plain` вообще уехал в несуществующий роман и выдуманное племя вместо ответа по тексту Дорошевича.
- В вопросе про Агасфера `plain` даёт общую легенду о Вечном Жиде, но не обязательно держится конкретных формулировок и акцентов данного текста.

### Где `rag` лучше

- На вопросе про аденские цистерны retrieval подаёт релевантный контекст, и модель отвечает в рамках корпуса.
- На вопросе про начало сюжета «Без аллаха» RAG уверенно возвращает завязку о том, что Аллаху надоело быть Аллахом и он спускается на землю.
- На вопросе про проклятие Агасфера `rag-structured` находит нужный фрагмент со словом «Иди!» и даёт содержательный ответ, в то время как `rag-fixed` промахивается.

## Вывод

Текущий результат уже показывает главную учебную идею RAG:

- без retrieval модель склонна уверенно достраивать ответ по ассоциациям;
- с retrieval модель заметно лучше ограничивается доступным контекстом;
- если retrieval промахивается, RAG тоже не спасает.

Практический вывод по текущей реализации:

- `structured` выглядит более перспективной стратегией для RAG, чем `fixed`;
- качество retrieval пока нестабильно и именно это главный резерв для следующего улучшения;
- даже в текущем виде система уже демонстрирует разницу между `plain` и `rag`, что и было основной целью этапа.
